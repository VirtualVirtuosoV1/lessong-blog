# Section 1: Introduction to Sequence Modeling (IMG_7080)

Traditional feedforward neural networks take a fixed-size input and produce a fixed-size output. Sequence modeling is crucial for data types where context and order are fundamental.

### 1.1 Taxonomy of Sequence Modeling Tasks
We can classify sequence problems based on the relationship between the lengths of input ($T_{in}$) and output ($T_{out}$) sequences.

1.  **One-to-One:** A standard neural network task. Input and output are fixed-size vectors.
    * *Example:* **Binary Classification** (e.g., classifying an image as cat or dog).
2.  **Many-to-One:** A sequence input maps to a single vector output.
    * *Example:* **Sentiment Classification** (classifying a whole sentence or paragraph as "positive" or "negative").
    * *Greek Note Conversion:* This includes converting regression problems into classification. For example, instead of predicting the exact future price of a stock (regression), we predict only whether it will go *up* or *down* (classification).
3.  **One-to-Many:** A single input generates a sequence output.
    * *Example:* **Image Captioning** (the note previously said image classification, which is one-to-one; captioning is the correct example. Input is one image, output is a sequence of words).
4.  **Many-to-Many:** Both input and output are sequences.
    * *Example:* **Machine Translation** (input is a sentence in French, output is a sentence in English, which may be of different length).

### 1.2 The Core Concept: Neurons with Recurrence
To handle sequences, we need memory. The simple feedforward equation is:
$$\hat{y}_t = f(x_t)$$
For sequences, this is insufficient. A recurrent neuron uses the **past memory** of its previous state, $h_{t-1}$, in addition to the current **input** vector, $x_t$, to generate the current **output** $\hat{y}_t$.

The updated model of a recurrent neuron is:
$$\hat{y}_t = f(x_t, h_{t-1})$$

### 1.3 Unrolling an RNN (Recurrent Neural Network)
When unrolled through time, an RNN passes its hidden state ($h$) from one time step to the next:
- Input $x_0 \to$ Cell $\to$ Hidden State $h_0$ $\to$ Output $\hat{y}_0$
- Input $x_1 \to$ Cell (receiving $h_0$) $\to$ Hidden State $h_1$ $\to$ Output $\hat{y}_1$
- Input $x_2 \to$ Cell (receiving $h_1$) $\to$ Hidden State $h_2$ $\to$ Output $\hat{y}_2$

*Memory Options:*
1.  **Direct State Passing:** Passes the individual hidden state from step to step (e.g., $h_0, h_1, \dots$).
2.  **Learnable Weight Aggregation:** The neural network can learn weights to aggregate information across the whole sequence. This allows the model to capture deeper dependencies, but faces the constraint of computational data volume.

# Section 2: Recurrent Neural Networks (RNNs) and Design Criteria (IMG_7081)

The key to an RNN’s efficiency is that it reuses the *same weight matrices* ($W_{xh}$, $W_{hh}$, $W_{hy}$) and bias vectors ($b_h$, $b_y$) at *every time step* of the sequence processing.

### 2.1 Complete RNN Formulation (TA Addition)
Let's define the actual state update equations that were incomplete in the notes:

The **Hidden State** at time step $t$ is updated as:
$$h_t = \text{tanh}(W_{hh}h_{t-1} + W_{xh}x_t + b_h)$$
-   Where $W_{hh}$ is the weight matrix from the previous hidden state, $W_{xh}$ is the weight matrix from the current input, $b_h$ is the bias for the hidden update, and 'tanh' is a non-linear activation function.

The **Output** at time step $t$ is calculated as:
$$\hat{y}_t = W_{hy}h_t + b_y$$
-   Where $W_{hy}$ is the weight matrix from the current hidden state to the output, and $b_y$ is the bias for the output.

### 2.2 Computation Graph and Parameter Sharing
The shared parameters ($W_{xh}$, $W_{hh}$, $W_{hy}$) are what make the network manageable across long sequences. However, this structure creates a complex dependency: the current output $\hat{y}_t$ and its associated loss $L_t$ do not depend only on the current input $x_t$, but also on all previous inputs ($x_{t-1}, x_{t-2}, \dots$) and all shared parameters, because $h_t$ itself is a result of this entire chain.

### 2.3 Design Criteria for Sequence Models
An efficient sequence modeling architecture must achieve the following goals:

1.  **Handle Variable-Length Sequences:** The model must process sequences without a fixed, predetermined length.
2.  **Track Long-Term Dependencies:** The network must be able to remember crucial information that appeared far back in the sequence.
    * *Greek Example:* A supermarket inventory model where predictions for today depend much more on data from exactly **1 year ago** (seasonal trends) than data from just **yesterday**.
3.  **Maintain Information About Order:** The specific sequence order matters. If we change the order of data (e.g., permute words in a sentence), the semantic meaning changes, and the model's weights must capture this difference.
4.  **Share Parameters Across the Sequence:** Crucial for scalability. As seen with RNNs, shared weights allow handling long sequences without an exploded number of parameters to learn.

# Section 3: Training RNNs and the Long-Term Dependency Problem (IMG_7082)

Standard RNNs have a major weakness that makes achieving criteria 2 (Long-Term Dependencies) very difficult in practice.

### 3.1 Backpropagation Through Time (BPTT)
To train an RNN, we use an algorithm called Backpropagation Through Time. We calculate the gradient of a loss function $L$ (often the sum of losses at all time steps) with respect to the shared weight matrices $W$.

Because of parameter sharing, the loss at any step (e.g., $L_2$) depends on the parameter $W$, but $W$ influenced $h_2$, which depended on $h_1$, which depended on $h_0$. We must use the chain rule to backpropagate gradients *backwards through time*. The partial gradient calculation looks like this:
$$\frac{\partial L_T}{\partial W} = \sum_{t=1}^T \frac{\partial L_T}{\partial \hat{y}_T} \frac{\partial \hat{y}_T}{\partial h_T} \left( \prod_{k=t+1}^T \frac{\partial h_k}{\partial h_{k-1}} \right) \frac{\partial h_t}{\partial W}$$

### 3.2 Standard RNN Gradient Flow and its Problems
The product term in the equation above is problematic. Let's look conceptually at how the gradient propagates through the chain: $L \to \hat{y}_t \to h_t \to h_{t-1} \to \dots \to h_0$.

1.  **Exploding Gradient:** If the gradients along the chain are, on average, larger than 1, multiplying them repeatedly will cause the total gradient to grow exponentially large. This leads to wildly unstable training. (Often solved by *gradient clipping*).
2.  **Vanishing Gradient:** This is the much more common problem. The derivative of the standard 'tanh' activation function used in RNN hidden states is between 0 and 1 (with a maximum of 1 at 0). Multiplying many fractions less than 1 repeatedly results in a gradient value that vanishes exponentially toward zero.

*The Consequence:* The vanishing gradient means the parameters for early time steps (e.g., the weights influencing $h_0$ to $h_{t-10}$) stop being updated. The network loses its "long-term memory" and behaves more like a fixed-window feedforward network, ignoring ancient but crucial information.

### 3.3 The Solution: Gating Mechanisms and LSTMs
To combat the Vanishing Gradient problem, we need more sophisticated neural mechanisms. Instead of simple neurons that always pass all their information, we use neurons with **gating mechanisms** (LSTMs and GRUs).
-   *The Core Idea:* Use specialized **gates** (which are small neural networks themselves) to *selectively* choose what information to add to or remove from the network's persistent memory.

# Section 4: Long Short-Term Memory (LSTM) Networks (IMG_7082 & IMG_7083)

LSTMs are an RNN variant that directly addresses the vanishing gradient problem. The key innovation is the introduction of a specialized **Cell State** ($C_t$) that acts as an "information highway," running parallel to the hidden state ($h_t$). This highway has minimal interactions, allowing information to persist over many steps. These interactions are managed by three trainable "gates."

### 4.1 LSTM Component Decomposition (TA Complete Formulas)

A full LSTM cell at time step $t$ processes the current input $x_t$, the previous hidden state $h_{t-1}$, and the previous cell state $C_{t-1}$.

*Note:* In the following formulas, $\sigma$ is the sigmoid activation function (squashing output to $[0, 1]$), which controls the "gates" (1 = completely open, 0 = completely closed). $\cdot$ denotes element-wise matrix multiplication (Hadamard product). The notations $i_t, j_t$ etc. from the notes are mapped here.

#### 1. The Forget Gate (IMG_7082)
This gate decides what information from the previous cell state ($C_{t-1}$) is no longer relevant and should be discarded. It computes:
$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$
- $f_t$ is a vector of values between 0 and 1, used later to multiply $C_{t-1}$.

#### 2. The Input/Ignore Gate and Candidate Input (IMG_7083)
This two-part mechanism decides what new information from the current time step ($x_t$, $h_{t-1}$) should be stored in the cell state.
- **Ignore/Input Gate:** Decides which values we will update:
  $$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$$
- **Candidate Input:** Creates a candidate vector of new information (this corresponds to $j_t$ or the input candidate $\tilde{C}_t$ from your list):
  $$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$$

#### 3. Update Cell State ($C_t$)
This is the core persistent memory update. We combine the "forgotten" past with the "selected" present information. This equation *replaces* the standard RNN's product chain with a combination of an element-wise multiplication (forget) and an element-wise addition (new information), which is highly resistant to vanishing gradients:
$$C_t = f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t$$

#### 4. The Output Gate and Hidden State ($h_t$) (IMG_7083)
This gate controls what information from the newly updated cell state ($C_t$) we will output and use as the visible hidden state ($h_t$) for the current time step and the next.
- **Output Gate:** Decides which parts of the state we will focus on for output:
  $$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$$
- **Hidden State Update:** We apply 'tanh' to the current cell state (pushing values to $[-1, 1]$) and then multiply it by the output gate's decision:
  $$h_t = o_t \cdot \tanh(C_t)$$

### 4.2 LSTM Variations and Training Concepts
-   **Gated Recurrent Units (GRUs):** A "simplified LSTM." It combines the forget and input gates into a single "update gate." It also merges the hidden state and cell state into a single state, $h_t$. This is simpler and often equally effective.
-   **Layered (Stacked) LSTMs:** We can increase model depth by stacking multiple LSTM layers, passing the hidden states $h_t^{(1)} \to h_t^{(2)} \to h_t^{(3)}$. The notes suggest not using too many, typically **2-3 layers**.
-   **Bidirectional LSTMs:** For processing the whole sequence at once. It runs one standard LSTM forward (from $t=0 \to T$) and another separate LSTM backward (from $t=T \to 0$). The hidden states are concatenated, allowing the model to "see" future context for any point $t$.
-   **Training Start Concepts:** (Conceptual reference)
    * **Warm Starts:** Initialize the recurrent states with informative values derived from prior data or layers.
    * **Cold Starts:** Initialize recurrent states with zeros or random values.
    * **When Losses Start:** Determining when to begin accumulating loss during BPTT, sometimes after a stabilization period.

# Section 5: The Limitations of LSTMs and Transition to Attention (IMG_7084)

LSTMs are powerful, but they have major limitations that modern models have sought to overcome.

### 5.1 Limitations
The fundamental flaw is inherent to all RNN architectures: **sequential dependency**. To calculate state $h_t$, we *must* already know $h_{t-1}$. This makes parallel processing of sequence elements impossible. RNNs are slow to train.

### 5.2 Desired Capabilities (Leading to Attention/Transformers)
- **Desired:** Handle continuous streams of data.
- **Desired:** Full parallelization of processing sequence elements.
- **Desired:** Robust, theoretical unlimited "long memory" that doesn't rely on vanishing or exploding gradients.

# Section 6: Understanding the Self-Attention Mechanism (IMG_7084)

The "Self-Attention" mechanism (pioneered by the Transformer architecture) completely bypasses recurrence. It looks at the whole sequence simultaneously and computes direct dependencies between *every* element and *every other* element. This solves the "long memory" constraint completely and allows for massive parallelization.

### 6.1 Conceptual Analogy: Search (Query, Key, Value)
We can understand attention with a database search analogy:
- **Query (Q):** What I am currently looking for (e.g., my current element's concept).
- **Key (K):** How other elements in the database are indexed or labeled (e.g., the title/labels of all other elements).
- **Value (V):** The actual content or information I receive once I find a match.

*Red Note Example:* You are searching for a video.
1.  **Search Query (Q):** "Machine Learning."
2.  **Database Titles (K):** Titles of thousands of videos, e.g., "Intro to PyTorch", "Stock Prediction with RNNs", "Understanding Transformers."
3.  **Matches (K's similar to Q):** The algorithm determines "Understanding Transformers" has a very high similarity to your query.
4.  **Result (V):** You are shown the actual content of the "Understanding Transformers" video, because that is its conceptual **Value**.
*The model's "value" is a function of the key it matched:* $V(K_{matched})$.

*Types:*
-   **Self-Attention:** When the Query, Key, and Value vectors are all derived from the *same* source sequence.
-   **Cross-Attention (Mentioned):** When the Query comes from one sequence (e.g., the target decoder sequence in translation) and the Key/Value vectors come from a different sequence (e.g., the source encoder sequence).

### 6.2 Deep Dive into Self-Attention Implementation

The steps to implement learnable self-attention are:

#### Step 1: Encode Position Information
Since self-attention processes all elements simultaneously, the model inherently *loses all information about order*. To fix this, we must inject an explicit "Positional Encoding" (like a timestamp or a sine/cosine wave pattern) into the input embedding of each word before the attention mechanism starts. This allows the model to differentiate between "dog bites man" and "man bites dog."

#### Step 2: Extract Query, Key, Value for Search
From the input embedding $X$, we perform three distinct linear projections (using three separate learnable weight matrices: $W_Q, W_K, W_V$ or *πινακάκια* as in your notes) to create $Q, K$, and $V$ vectors for every element:
$$Q = X W_Q$$
$$K = X W_K$$
$$V = X W_V$$

#### Step 3: Compute Attention Weighting
This is how the "fast way: inner product" is used. We take the Query vector of my current focus and compute the inner (dot) product with the Key vectors of *every other element* in the sequence. A high dot product means high semantic similarity/attention. We normalize these raw scores into probabilities.

*Note Conversion:* In the scaling, the factor in the denominator is usually $\sqrt{d_k}$, where $d_k$ is the dimensionality of the key vectors. This prevents the scores from growing too large.

We apply 'softmax' across the sequence to get normalized weights that sum to 1:
$$\text{softmax}\left( \frac{Q K^T}{\sqrt{d_k}} \right)$$

#### Step 4: Extract Features with High Attention (TA Complete Formula)
Finally, we use these probabilities to create a weighted average of the original **Value** vectors ($V$). This means my current focus position ($Q$) pays *most attention* to (and incorporates most semantic information from) the values ($V_j$) whose keys ($K_j$) were most similar to my query.

The full self-attention formula (often called **Scaled Dot-Product Attention**) is:
$$\text{Attention}(Q, K, V) = \text{softmax}\left( \frac{Q K^T}{\sqrt{d_k}} \right) V$$

### 6.3 Multi-Head Attention and Multimodal Inputs
-   **Multi-head Attention:** A single self-attention mechanism might focus on one semantic aspect (e.g., grammatical structure). By having multiple parallel self-attention "heads" (each with their own learnable $W_{Q,K,V}$ matrices), the model can learn different, diverse aspects of semantic dependency simultaneously.
-   **These operations form a self-attention head that can plug into** larger Transformer blocks.
-   **Multimodal Inputs:** (Greek Note) If the input consists of different types of data, e.g., both image patches and text embeddings, we must first vectorize and project them all into a common conceptual vector space and dimensional depth. This brings them "ε 1 διάσταση" (into 1 unified vector space) before concatenating them into a single, long sequence vector for the self-attention mechanism to process.
