# Αξιολόγηση Μοντέλων: Τεχνικές Διαχωρισμού και Μετρικές Επίδοσης

> **💡 Σημείωση TA (Βοηθού Καθηγητή):** Στο σημερινό μάθημα εστιάζουμε στο *πώς* καταλαβαίνουμε αν ένα μοντέλο που εκπαιδεύσαμε είναι όντως καλό. Η διαδικασία αυτή είναι κρίσιμη, διότι ένα μοντέλο μπορεί να "αποστηθίσει" τα δεδομένα αντί να μάθει τη γενική δομή τους (το γνωστό πρόβλημα του Overfitting).
> *(Υπενθύμιση: Έχει ανακοινωθεί η 2η εργασία με προθεσμία παράδοσης μετά το Πάσχα, 3/5).*

## 1. Αξιολόγηση στο Σύνολο Εκπαίδευσης (Training Set)

Η πιο απλοϊκή προσέγγιση είναι ο υπολογισμός της επίδοσης ενός μοντέλου στα **ίδια δεδομένα** από τα οποία εκπαιδεύτηκε (training set).
* Το σφάλμα που προκύπτει όταν μετράμε την απόδοση στα δεδομένα εκπαίδευσης καλείται **σφάλμα επανακατάστασης (resubstitution error)**.
* **⚠️ Σημαντικό:** Αυτή η πρακτική είναι λανθασμένη για την αξιολόγηση, καθώς δεν μας δείχνει την ικανότητα του μοντέλου να γενικεύει σε νέα, άγνωστα δεδομένα.

### Διαδικασία Εκπαίδευσης και Πρόβλεψης
Συνήθως, διαχωρίζουμε τα δεδομένα μας (πίνακας χαρακτηριστικών $X$ και διάνυσμα στόχου $y$) σε υποσύνολα:
1. **$X_{train}, y_{train}$**: Για την εκπαίδευση του αλγορίθμου $\rightarrow$ `fit(X_train, y_train)`
2. **$X_{val}, y_{val}$**: Για την επικύρωση και επιλογή υπερπαραμέτρων.
3. **$X_{test}, y_{test}$**: Για την τελική αξιολόγηση $\rightarrow$ $\hat{y} = \text{predict}(X_{test})$

### Παράδειγμα Πολυπλοκότητας Μοντέλου (Άσκηση)
Έχουμε ένα σύνολο 1000 δειγμάτων το οποίο χωρίζουμε σε 800 για εκπαίδευση και 200 για έλεγχο. Έστω ότι δοκιμάζουμε πολυωνυμικά μοντέλα διαφορετικού βαθμού ($d = 2 \dots 10$).
* $d=2$: $75\%$ Accuracy
* $d=3$: $85\%$ Accuracy
* **$d=4$: $87\%$ Accuracy (Βέλτιστο)**
* $d=10$: $78\%$ Accuracy

> **💡 Επεξήγηση TA:** Βλέπουμε ότι καθώς αυξάνεται η πολυπλοκότητα (βαθμός $d$), η ακρίβεια αρχικά βελτιώνεται. Όταν όμως φτάσουμε στο $d=10$, η ακρίβεια στο test set **πέφτει** στο 78%. Αυτό είναι κλασικό παράδειγμα **Υπερεκπαίδευσης (Overfitting)**. Το μοντέλο με $d=4$ έχει την καλύτερη ικανότητα γενίκευσης.

## 2. Μέθοδοι Αξιολόγησης και Διαχωρισμού Δεδομένων

### Επαναλαμβανόμενη Κράτηση (Repeated Holdout)
Η απλή **μέθοδος της κράτησης (holdout)** χωρίζει το dataset μια φορά (π.χ. 80-20). Για πιο αξιόπιστα αποτελέσματα κάνουμε **επαναλαμβανόμενη κράτηση**, δηλαδή τρέχουμε το holdout πολλές φορές με διαφορετικούς τυχαίους διαχωρισμούς.
* **Μειονέκτημα:** Οι εκτιμήσεις δεν είναι μεταξύ τους ανεξάρτητες (κάποια δείγματα μπορεί να βρεθούν στο test set πολλές φορές και κάποια καμία).
* **Λύση $\rightarrow$ K-fold Cross-Validation:** Χωρίζουμε τα δεδομένα σε $k$ ίσα μέρη. Το μοντέλο εκπαιδεύεται $k$ φορές, χρησιμοποιώντας κάθε φορά ένα διαφορετικό fold ως test set και τα υπόλοιπα $k-1$ ως training set.
* *Σημείωση για Νευρωνικά Δίκτυα (Deep Learning):* Η μέθοδος K-fold **δεν** προτιμάται στα νευρωνικά δίκτυα επειδή απαιτεί πολλαπλές εκπαιδεύσεις, κάτι που έχει τεράστιο υπολογιστικό κόστος. Επιπλέον, στα Deep Learning προβλήματα έχουμε συνήθως τόσο μεγάλα datasets που ένα απλό Holdout (με ένα μεγάλο, αντιπροσωπευτικό test set) είναι αρκετό.

### Πολύ Μικρό Πλήθος Δεδομένων: Leave-One-Out CV
Όταν έχουμε ελάχιστα δεδομένα, χρησιμοποιούμε **Σταυρωτή επικύρωση "άφησε ένα εκτός" (Leave-One-Out Cross-Validation - LOOCV)**.
* Σε ένα dataset μεγέθους $m$, κάνουμε **$m$ επαναλήψεις**.
* Σε κάθε επανάληψη, **1 παράδειγμα** γίνεται το σύνολο ελέγχου (test) και τα υπόλοιπα **$m-1$** γίνονται το σύνολο εκπαίδευσης.
* **Πλεονεκτήματα:** Μεγιστοποιεί τη χρήση των δεδομένων για εκπαίδευση, είναι πλήρως ντετερμινιστική μέθοδος (δεν έχει τυχαιότητα).
* **Μειονέκτημα:** Εξαιρετικά υψηλό υπολογιστικό κόστος (πρέπει να εκπαιδεύσεις $m$ ξεχωριστά μοντέλα).

### Μέθοδος Bootstrap (Με ίδιες δυνάμεις)
* Γίνεται **δειγματοληψία με επανατοποθέτηση (sampling with replacement)** $m$ δεδομένων για τη δημιουργία του training set. Κάποια δείγματα θα επιλεγούν πολλές φορές, ενώ κάποια δεν θα επιλεγούν καθόλου (αυτά ονομάζονται *Out-of-Bag* δείγματα και λειτουργούν ως test set).
* Ο τελικός υπολογισμός του σφάλματος συνδυάζει την επίδοση στα μη επιλεγμένα δεδομένα ($e_{test}$) και στα επιλεγμένα ($e_{train}$):
  $$E_{bootstrap} = 0.632 \cdot e_{test} + 0.368 \cdot e_{train}$$
  > **💡 Επεξήγηση TA (Γιατί 0.632;):** Μαθηματικά, όταν επιλέγεις με επανατοποθέτηση $N$ φορές από $N$ δείγματα, η πιθανότητα ένα συγκεκριμένο δείγμα να **μην** επιλεγεί ποτέ προσεγγίζει το $1/e \approx 0.368$. Άρα, το $63.2\%$ των μοναδικών δειγμάτων καταλήγουν στο training set και το $36.8\%$ μένουν απ' έξω (test set).

## 3. Ειδικές Τεχνικές Διαχωρισμού

### Στρωματοποίηση (Stratification)
Αποτελεί τον διαχωρισμό δεδομένων ταξινόμησης (classification) **με βάση την κλάση**, ώστε η κατανομή των κλάσεων στα υποσύνολα (train/test) να είναι **η ίδια με το αρχικό σύνολο**.
* Π.χ., αν στο dataset το 90% είναι "Υγιείς" και το 10% "Ασθενείς", η στρωματοποίηση εξασφαλίζει ότι και το test set θα έχει την ίδια αναλογία 9:1.
* Συνήθως βελτιώνει (και σταθεροποιεί) τα αποτελέσματα της αξιολόγησης.

### Ομαδοποίηση (Group-based Split)
Χρησιμοποιείται για πιο ρεαλιστικό διαχωρισμό με βάση **κάποια μεταβλητή του πεδίου** (grouping variable), προκειμένου να αποφύγουμε τη διαρροή πληροφορίας (data leakage).
* **Παράδειγμα:** Έστω ότι έχουμε δεδομένα από μαγαζιά σε διάφορες πόλεις και το μοντέλο μας πρέπει να προβλέψει αν θα πετύχει το άνοιγμα ενός νέου μαγαζιού σε μια **νέα πόλη**.
* Αν χωρίσουμε τυχαία, μαγαζιά από την *ίδια* πόλη θα βρεθούν και στο train και στο test set. Έτσι το μοντέλο ίσως απλά "μάθει" τα χαρακτηριστικά της συγκεκριμένης πόλης και όχι να γενικεύει.
* **Λύση:** Ομαδοποιούμε ανά πόλη. Αν μια πόλη είναι στο train set, κανένα μαγαζί της δεν μπαίνει στο test set (πρακτικά μιλάμε για το *GroupKFold* του scikit-learn).

### Εκπαίδευση για Παραγωγή (Production)
* Τι κάνουμε όταν βρούμε το βέλτιστο μοντέλο μέσω Validation/Test splits;
* Στην πράξη, μπορούμε να επανεκπαιδεύσουμε το μοντέλο **στην ένωση των συνόλων εκπαίδευσης και επικύρωσης** (ή σε όλα τα διαθέσιμα δεδομένα) πριν το βγάλουμε σε παραγωγικό περιβάλλον (production) με άγνωστα δεδομένα.
* **Εξαίρεση:** Αυτό ΔΕΝ συνιστάται σε "ασταθείς" αλγόριθμους, όπως τα μικρά νευρωνικά δίκτυα, όπου η προσθήκη έστω και λίγων νέων δεδομένων μπορεί να αλλάξει δραματικά (και απρόβλεπτα) τα βάρη του τελικού μοντέλου.

## 4. Μετρικές Αξιολόγησης (Evaluation Metrics)

### Μετρικές Παλινδρόμησης (Regression)
Στην παλινδρόμηση, όπου προβλέπουμε συνεχείς τιμές, χρησιμοποιούμε κλασσικές μετρικές απόστασης / σφάλματος:
* **MAE (Mean Absolute Error):** Μέσο Απόλυτο Σφάλμα.
  $$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$
* **MSE (Mean Squared Error):** Μέσο Τετραγωνικό Σφάλμα (τιμωρεί περισσότερο τα μεγάλα λάθη).
  $$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$
*(Όπου $n$ το πλήθος των δειγμάτων, $D$ οι διαστάσεις, $y$ η πραγματική τιμή και $\hat{y}$ η πρόβλεψη).*

### Μετρικές Ταξινόμησης (Classification)
Στην ταξινόμηση, όλα ξεκινάνε από τον **Πίνακα Σύγχυσης (Confusion Matrix)**. 
Ο πίνακας αυτός συγκρίνει την **Πραγματικότητα** (True / False - ανήκει ή δεν ανήκει στην κλάση) με την **Πρόβλεψη** του μοντέλου (Positive / Negative).

| | Πραγματικότητα: Θετικό (True) | Πραγματικότητα: Αρνητικό (False) |
| :--- | :---: | :---: |
| **Πρόβλεψη: Θετικό (Positive)** | True Positive (**TP**) | False Positive (**FP**) |
| **Πρόβλεψη: Αρνητικό (Negative)** | False Negative (**FN**) | True Negative (**TN**) |

Από τον πίνακα αυτόν προκύπτουν οι βασικές μετρικές (Όροι στα Ελληνικά):
1. **Ορθότητα (Accuracy):** Πόσες προβλέψεις (θετικές και αρνητικές) κάναμε σωστά στο σύνολο.
   $$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$
2. **Ακρίβεια (Precision):** Από όσα *προβλέψαμε* ως θετικά, πόσα ήταν όντως θετικά.
   $$\text{Precision} = \frac{TP}{TP + FP}$$
3. **Ανάκληση / Ευαισθησία (Recall / Sensitivity):** Από όσα *είναι στην πραγματικότητα* θετικά, πόσα καταφέραμε να "ανακαλέσουμε" (να εντοπίσουμε).
   $$\text{Recall} = \frac{TP}{TP + FN}$$
