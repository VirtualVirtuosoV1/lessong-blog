## Εισαγωγή στο Deep Learning & AI
* Το **Deep Learning** (Βαθιά Μηχανική Μάθηση) είναι μια υποκατηγορία της Μηχανικής Μάθησης (Machine Learning) και κατ' επέκταση της Τεχνητής Νοημοσύνης (Artificial Intelligence).
* Αποτελεί μια τεχνική ML που μαθαίνει χαρακτηριστικά (features) και εργασίες **απευθείας από τα δεδομένα**.
* Η εκμάθηση των χαρακτηριστικών γίνεται μέσω **Νευρωνικών Δικτύων (Neural Networks)**.

**Γιατί το Deep Learning γνωρίζει τεράστια άνθιση τώρα;**
1. **Τεράστιος όγκος δεδομένων (Massive amount of data)**: Ζούμε στην εποχή των Big Data, τα οποία είναι απαραίτητα για την εκπαίδευση αυτών των μοντέλων.
2. **Βελτιωμένο hardware**: Η σύγχρονη υπολογιστική ισχύς επιτρέπει την επεξεργασία αυτού του τεράστιου όγκου δεδομένων.
3. **Νέα αρχιτεκτονική λογισμικού (New software architecture)**: Σύγχρονα εργαλεία και βιβλιοθήκες έχουν απλοποιήσει την ανάπτυξη.

## Νευρωνικά Δίκτυα (Neural Networks)
* Είναι εμπνευσμένα από τον ανθρώπινο εγκέφαλο.
* Το βασικό δομικό στοιχείο (building block) ενός Νευρωνικού Δικτύου είναι ο **Νευρώνας (Neuron)**. * **Διαδικασία**: Δίνουμε δεδομένα (Data input) $\rightarrow$ Το δίκτυο εκπαιδεύεται (train) για την κατανόηση προτύπων (patterns) $\rightarrow$ Εξάγει χρήσιμες προβλέψεις (output useful predictions).

### Η διαδικασία μάθησης ενός Νευρωνικού Δικτύου
Η μάθηση χωρίζεται σε δύο κύρια στάδια:
1. **Forward Propagation (Εμπρόσθια διάδοση)**: Η πληροφορία ταξιδεύει από το επίπεδο εισόδου (input layers) προς τα κρυφά επίπεδα (hidden layers) και τελικά στην έξοδο.
2. **Backward Propagation / Backpropagation (Οπίσθια διάδοση)**: Η πληροφορία ταξιδεύει αντίστροφα, από το επίπεδο εξόδου (output layer) πίσω στα κρυφά επίπεδα (hidden layers) για να προσαρμόσει τις παραμέτρους.

* **Loss Function (Συνάρτηση Απώλειας)**: Βοηθά να ποσοτικοποιηθεί η απόκλιση (deviation) της πρόβλεψης από το αναμενόμενο αποτέλεσμα.
* **Βάρος (Weight)**: Καθορίζει πόσο "σημαντικός" είναι ένας νευρώνας στη σχέση του με τους επόμενους.
* **Πόλωση (Bias)**: Λειτουργεί σαν την "άποψη" του νευρώνα. Επιτρέπει τη μετατόπιση της συνάρτησης ενεργοποίησης προς τα δεξιά ή προς τα αριστερά.

*Σημείωση του TA: Στην αρχή, τα βάρη και τα biases αρχικοποιούνται με τυχαίες τιμές. Το δίκτυο επαναλαμβάνει την οπίσθια διάδοση (backpropagation) συνεχώς, μέχρι να ελαχιστοποιηθεί το σφάλμα (loss).*

## Συναρτήσεις Ενεργοποίησης (Activation Functions)
* Αποφασίζουν εάν ένας νευρώνας θα **ενεργοποιηθεί** (δηλαδή αν θα συνεισφέρει στο επόμενο επίπεδο).

### 1. Step Function (Συνάρτηση Βήματος)
* Λειτουργεί με λογική κατωφλίου (threshold): Αν η τιμή (value) είναι μεγαλύτερη από ένα όριο, ενεργοποιείται. Αλλιώς, δεν ενεργοποιείται.
* **Μειονέκτημα**: Δεν μπορεί να μοντελοποιήσει κάθε κατάσταση (π.χ. αν έχουμε περισσότερες από 2 κλάσεις και ενεργοποιηθούν πολλαπλοί νευρώνες, είναι δύσκολο να αποφασίσουμε ποια είναι η σωστή).

### 2. Linear Function (Γραμμική Συνάρτηση)
* Η παράγωγος (derivative) της είναι μια σταθερά (constant).
* **Μειονέκτημα**: Αν χρησιμοποιήσουμε μόνο γραμμικές συναρτήσεις, όλα τα κρυφά επίπεδα (hidden layers) θα μπορούσαν να συμπτυχθούν μαθηματικά στο πρώτο επίπεδο (Κακό!). Γι' αυτό χρειαζόμαστε **Μη Γραμμικές (Non-Linear)** συναρτήσεις, ώστε να μοντελοποιήσουμε κάθε τύπο δεδομένων χωρίς περιορισμούς.

### 3. Sigmoid Function (Σιγμοειδής)
* Μαθηματικός τύπος: $\sigma(x) = \frac{1}{1 + e^{-x}} \in (0,1)$.
* **Πλεονεκτήματα**: Είναι μη γραμμική (Non-linear) και παράγει αναλογικές (analog) εξόδους.
* **Μειονεκτήματα**: Υπάρχουν δραστικές αλλαγές μόνο όταν το $x \in (-2,2)$. Στα άκρα υποφέρει από το **Vanishing Gradient Problem** (Πρόβλημα Εξαφανιζόμενης Κλίσης).

### 4. Tanh Function (Υπερβολική Εφαπτομένη)
* Μαθηματικός τύπος: $\tanh(x) = \frac{2}{1 + e^{-2x}} - 1 = 2\sigma(2x) - 1$.
* **Πλεονεκτήματα**: Μη γραμμική, με αναλογικές έξοδοι. Είναι πιο ομαλή, αλλά με πιο απότομη παράγωγο σε σχέση με την Sigmoid.
* **Μειονέκτημα**: Επίσης υποφέρει από το **Vanishing Gradient problem**.
  * *Ορισμός Vanishing Gradient*: Καθώς η κλίση (gradient) συρρικνώνεται επίπεδο-προς-επίπεδο κατά τη διάρκεια του backpropagation, τα πρώτα επίπεδα δυσκολεύονται να μάθουν αποτελεσματικά, επιβραδύνοντας ή και σταματώντας εντελώς την εκπαίδευση.

### 5. ReLU Function (Rectified Linear Unit)
* Μαθηματικός τύπος: $R(x) = \max(0, x)$. Το εύρος της είναι από $0$ έως $\infty$. * **Πλεονεκτήματα**:
  * Μοιάζει, αλλά **δεν** είναι γραμμική.
  * Προσφέρει αραιές ενεργοποιήσεις (Sparse activations). Μόνο το $50\%$ των νευρώνων πυροδοτεί (fire), κάνοντας το δίκτυο πιο αποδοτικό.
  * Είναι πολύ λιγότερο υπολογιστικά ακριβή σε σχέση με την tanh.
* **Μειονεκτήματα**:
  * Η κλίση (gradient) είναι $0$ για $x < 0$. Αυτό σημαίνει ότι στο backpropagation δεν αλλάζει τίποτα.
  * **Dying ReLU problem**: Κάποιοι νευρώνες απλά "πεθαίνουν" και σταματούν να μαθαίνουν.
  * *Λύση*: **Leaky ReLU**. Προσθέτει μια πολύ μικρή κλίση (slope) για τις αρνητικές τιμές, ώστε να μην μηδενίζεται εντελώς το gradient.

**Ποια να χρησιμοποιήσεις;**
Εξαρτάται. Αν η συνάρτηση που προσπαθείς να προσεγγίσεις έχει συγκεκριμένα χαρακτηριστικά, διάλεξε αυτή που θα την προσεγγίσει γρηγορότερα. 
* Κανόνας: Χρησιμοποίησε **ReLU** όταν δεν είσαι σίγουρος.
* Για προβλήματα δυαδικής ταξινόμησης (Binary classification): Χρησιμοποίησε **Sigmoid**.

## Συναρτήσεις Απώλειας (Loss Functions)
Ποσοτικοποιούν το σφάλμα. Επιλέγουμε ανάλογα με τις ανάγκες του προβλήματος:
1. **Regression (Παλινδρόμηση)**: Squared error, Huber loss.
2. **Binary Classification (Δυαδική Ταξινόμηση)**: Binary cross entropy, Hinge loss.
3. **Multi-class classification (Πολυταξική Ταξινόμηση)**: Multi-class cross-entropy, Kullback-Leibler divergence.

## Optimizers (Βελτιστοποιητές)
Κατά την εκπαίδευση, ρυθμίζουμε τις παραμέτρους για να ελαχιστοποιήσουμε τη συνάρτηση απώλειας και να κάνουμε το μοντέλο μας όσο το δυνατόν πιο βελτιστοποιημένο (optimised). Οι optimizers συνδέουν τη loss function και τις παραμέτρους του μοντέλου ενημερώνοντας το δίκτυο.

### Gradient Descent (Κάθοδος Κλίσης - "Ο παππούς των optimizers")
Αλγόριθμος επανάληψης που ξεκινά από ένα τυχαίο σημείο της loss function και ταξιδεύει προς τα κάτω στην πλαγιά (σε βήματα) μέχρι να φτάσει στο χαμηλότερο σημείο (minimum) της συνάρτησης. Είναι πολύ δημοφιλής, γρήγορος, ανθεκτικός (robust) και ευέλικτος.

**Βήματα:**
1. Υπολογισμός του τι θα έκανε μια μικρή αλλαγή σε κάθε μεμονωμένο βάρος στη loss function.
2. Προσαρμογή κάθε παραμέτρου βάσει της κλίσης (gradient) της.
3. Επανάληψη των βημάτων 1 & 2 μέχρι να ελαχιστοποιηθεί η loss function.

* Η Κλίση (Gradient) μιας συνάρτησης είναι το διάνυσμα των μερικών παραγώγων ως προς όλες τις ανεξάρτητες μεταβλητές: $\nabla f(x,y) = \left( \frac{\partial f}{\partial x}(x,y), \frac{\partial f}{\partial y}(x,y) \right)$.

**Learning Rate (Ρυθμός Μάθησης):**
Στο Gradient Descent ιδανικά θέλουμε να βρούμε το Ολικό Ελάχιστο (global minimum). 
* Η πολύ γρήγορη αλλαγή των βαρών μπορεί να εμποδίσει την ικανότητα ελαχιστοποίησης (π.χ. προσπερνάς το σωστό σημείο). Γι' αυτό χρησιμοποιούμε το Learning Rate.
* Συνήθως είναι ένας μικρός αριθμός (π.χ. 0.001) που πολλαπλασιάζεται για να κλιμακώσει τα gradients.
* Αν είναι **πολύ μικρό**: χρειάζεται περισσότερος χρόνος (more time) ή κολλάς σε τοπικό ελάχιστο (local minimum).

### Άλλοι Optimizers βασισμένοι στο Gradient Descent (GD)
* **Stochastic Gradient Descent (SGD)**: Υλοποίηση του GD που χρησιμοποιεί ένα υποσύνολο (subset) ή batches παραδειγμάτων εκπαίδευσης (αντί για ολόκληρο το dataset) σε κάθε πέρασμα. Ενσωματώνει τη δυναμική (momentum) συσσωρεύοντας gradients και είναι λιγότερο απαιτητικό υπολογιστικά.
* **Adagrad**: Προσαρμόζει το Learning Rate σε μεμονωμένα features (καλό για missing examples).
* **RMSprop** (από Adagrad): Συσσωρεύει gradients σε καθορισμένο παράθυρο (defined window).
* **Adadelta** (Σημείωση: "Adaprop"): Παρόμοιο με το RMSprop.
* **Adam (Adaptive Momentum Estimation)**: Χρησιμοποιεί momentum και είναι εξαιρετικά διαδεδομένο.

*Σημείωση: Το Backpropagation είναι ουσιαστικά το GD εφαρμοσμένο σε ένα Δίκτυο.*

## Parameters vs. Hyperparameters
* **Model Parameters (Παράμετροι Μοντέλου)**: Μεταβλητές εσωτερικές στο νευρωνικό δίκτυο (π.χ. weights, biases). Οι τιμές τους εκτιμώνται απευθείας από τα δεδομένα και απαιτούνται από το μοντέλο για να κάνει προβλέψεις. Ορίζουν τις ικανότητες (skill) του μοντέλου.
* **Model Hyperparameters (Υπερπαράμετροι Μοντέλου)**: Ρυθμίσεις (configuration) εξωτερικές στο νευρωνικό δίκτυο (π.χ. Learning rate, K στους αλγόριθμους KNN, C στα SVMs). Οι τιμές τους δεν μπορούν να εκτιμηθούν από τα δεδομένα και δεν υπάρχει ξεκάθαρος τρόπος να βρεις την καλύτερη τιμή. Όταν βελτιστοποιείς (tune) έναν αλγόριθμο ML, ρυθμίζεις τις υπερπαραμέτρους του. *Αν πρέπει να το κάνεις χειροκίνητα, είναι μάλλον hyperparameter.*.

## Epochs, Batches, Batch Size & Iterations
Τα μεγάλα datasets καθιστούν αυτούς τους όρους απαραίτητους. Χωρίζουμε τα μεγάλα datasets σε μικρότερα **batches** και τα τροφοδοτούμε στο νευρωνικό δίκτυο.
* **Epoch (Εποχή)**: Όταν ολόκληρο το dataset περάσει forward και backward μέσα από το δίκτυο **ακριβώς μία φορά**. Πολλαπλές εποχές βοηθούν στην καλύτερη γενίκευση, αλλά πάρα πολλές εποχές οδηγούν σε overfitting.
* **Batch size**: Ο συνολικός αριθμός των training examples σε ένα batch.
* **Iterations (Επαναλήψεις)**: Αριθμός από batches = Αριθμός επαναλήψεων σε μία εποχή.

## Τύποι Μηχανικής Μάθησης (Types of Learning)

### 1. Supervised Learning (Επιβλεπόμενη Μάθηση)
* Οι αλγόριθμοι μαθαίνουν μέσω παραδειγμάτων χρησιμοποιώντας **καλά επισημειωμένα (well-labeled)** δεδομένα.
* Κάθε παράδειγμα αποτελείται από μία είσοδο (input - τυπικά ένα διάνυσμα) και την επιθυμητή έξοδο (desired output).
* Στόχος: Η πρόβλεψη της σωστής ετικέτας (label) για τα δεδομένα. Στην πιο απλή μορφή: $y = f(x)$.
* Χωρίζεται σε:
  * **Classification (Ταξινόμηση)**: Το μοντέλο βρίσκει χαρακτηριστικά στα δεδομένα που συσχετίζονται με μια κλάση και δημιουργεί μια συνάρτηση χαρτογράφησης (mapping function). Παραδείγματα: Linear classifiers, Support Vector Machines (SVMs), Random Forest.
  * **Regression (Παλινδρόμηση)**: Το μοντέλο προσπαθεί να βρει τη σχέση μεταξύ εξαρτημένων και ανεξάρτητων μεταβλητών. Στόχος: πρόβλεψη συνεχών (continuous) τιμών. Παραδείγματα: Linear regression, Lasso regression, Multivariate regression.
    * Βασική εξίσωση απλής γραμμικής παλινδρόμησης (ένα χαρακτηριστικό εισόδου): $\hat{y} = wx + b$ (όπου $x$: input feature, $w$: slope/βάρος, $b$: y-intercept/πόλωση).
    * Εξίσωση για πολλά χαρακτηριστικά: $\hat{y} = w[0]\cdot x[0] + w[1]\cdot x[1] + \dots + w[i]\cdot x[i] + b$.

### 2. Unsupervised Learning (Μη Επιβλεπόμενη Μάθηση)
* Χρησιμοποιείται για να φανερώσει υποκείμενα μοτίβα (underlying patterns) σε **μη επισημειωμένα (unlabelled)** δεδομένα. Βασίζεται αποκλειστικά στα χαρακτηριστικά (features) των δεδομένων.
* Χωρίζεται σε:
  * **Clustering (Ομαδοποίηση)**: Η διαδικασία ομαδοποίησης δεδομένων σε διαφορετικά clusters/groups. Μπορεί να είναι Partitional (κάθε αντικείμενο ανήκει σε 1 κλάση) ή Hierarchical (μπορούν να αναπαρασταθούν σε διαγράμματα δέντρων - tree diagrams). Αλγόριθμοι: K-means, Expectation Maximization, Hierarchical Cluster Analysis.
  * **Association (Συσχέτιση)**: Προσπαθεί να βρει σχέσεις μεταξύ διαφορετικών οντοτήτων. 
* Εφαρμογές: Συστήματα προτάσεων AirBnb & Amazon, Ανίχνευση απάτης πιστωτικών καρτών (Credit card fraud detection).

### 3. Reinforcement Learning (Ενισχυτική Μάθηση)
* Επιτρέπει σε έναν "πράκτορα" (agent) να μαθαίνει σε ένα διαδραστικό περιβάλλον (interactive environment) μέσω δοκιμής και πλάνης (trial & error).
* Βασίζεται σε ανατροφοδότηση από τις δικές του ενέργειες, χρησιμοποιώντας ανταμοιβές και τιμωρίες (rewards & punishments) ως σήματα για θετική ή αρνητική συμπεριφορά.
* Στόχος: Εύρεση ενός κατάλληλου μοντέλου που μεγιστοποιεί τη συνολική αθροιστική ανταμοιβή (Goal oriented algorithms).
* Συνήθως μοντελοποιείται ως "Markov Decision Process" (Περιβάλλον $\rightarrow$ Κατάσταση (State) $\rightarrow$ Πράκτορας (Agent) $\rightarrow$ Ενέργεια (Action) $\rightarrow$ Ανταμοιβή (Reward)).
* Εφαρμογές: Ρομποτική, Στρατηγικός σχεδιασμός επιχειρήσεων, Έλεγχος φαναριών κυκλοφορίας, Web System Control.

## Regularization (Κανονικοποίηση)
Το κεντρικό πρόβλημα στο Deep Learning είναι το **Overfitting** (όταν το μοντέλο αποστηθίζει τα δεδομένα εκπαίδευσης και αδυνατεί να γενικεύσει σε νέα δεδομένα). Λύσεις (Solutions-ish):
1. **Dropout**: Αφαιρεί τυχαία (randomly removes) μερικούς κόμβους (nodes) και τις συνδέσεις τους, εμποδίζοντας την υπερβολική εξάρτηση μεταξύ των νευρώνων.
2. **Data Augmentation**: Περισσότερα δεδομένα = Καλύτερο Μοντέλο. Εφαρμόζουμε μετασχηματισμούς (transformations) στο υπάρχον dataset για να δημιουργήσουμε (synthesize) "ψεύτικα" (fake) νέα δεδομένα.
3. **Early Stopping**: Κατά την εκπαίδευση, το training error μειώνεται σταθερά, αλλά το validation error μπορεί να αρχίσει να αυξάνεται μετά από ένα ορισμένο σημείο. Εκεί σταματάμε την εκπαίδευση.

## Αρχιτεκτονικές Νευρωνικών Δικτύων (Neural Network Architectures)

### 1. Fully-connected Feed Forward Neural Networks
* Κάθε νευρώνας συνδέεται με κάθε επόμενο επίπεδο χωρίς καμία σύνδεση προς τα πίσω (no backward connections/loops).
* Λαμβάνουν μια είσοδο σταθερού μεγέθους (fixed-size input) και επιστρέφουν μια έξοδο σταθερού μεγέθους.
* Υπάρχουν αναρίθμητοι συνδυασμοί: διαφορετικά inputs, outputs, hidden layers, αριθμός νευρώνων, activation functions.
* **Μειονέκτημα**: Δεν μπορούν να μοντελοποιήσουν ακολουθιακά δεδομένα.

### 2. Recurrent Neural Networks (RNNs - Αναδρομικά)
* Τα απλά (Vanilla) NNs δεν μπορούν να χειριστούν ακολουθιακά δεδομένα (sequential data) επειδή δεν μοιράζονται παραμέτρους μέσα στον χρόνο. Η κοινοχρησία παραμέτρων (sharing parameters) επιτρέπει στο δίκτυο να αναζητά ένα χαρακτηριστικό παντού στην ακολουθία, και όχι μόνο σε μια συγκεκριμένη περιοχή.
* Τα RNNs χρησιμοποιούν έναν **βρόχο ανατροφοδότησης (feedback loop)** στα κρυφά επίπεδα.
* Αυτό δίνει στο μοντέλο τη δυνατότητα "βραχυπρόθεσμης μνήμης" (short-term memory).
* Εκπαιδεύονται με αλγορίθμους Backpropagation εφαρμοζόμενους για κάθε σημείο της ακολουθίας, τεχνική γνωστή ως **Backpropagation Through Time (BPTT)**.
* **Πρόβλημα Vanishing Gradient Problem (VGP)**: Η βραχυπρόθεσμη μνήμη των RNNs οφείλεται στο VGP, το οποίο προκύπτει από τη φύση του Backpropagation. Η εξίσωση ανανέωσης είναι $W = W + \eta \frac{dE}{dW}$. Τα gradients ενός επιπέδου υπολογίζονται βάσει των gradients των προηγούμενων επιπέδων. Αν η αρχική προσαρμογή του gradient είναι μικρή, οι προσαρμογές στα επόμενα επίπεδα θα είναι μικρότερες, οδηγώντας σε vanishing gradients.
* Λόγω του VGP, τα απλά RNNs αδυνατούν να μάθουν εξαρτήσεις μεγάλης εμβέλειας (long-range dependencies).
* **Λύση - LSTMs (Long Short-Term Memory) & Gated RNNs (όπως GRUs)**: Είναι ικανά να μαθαίνουν μακροπρόθεσμες εξαρτήσεις χρησιμοποιώντας "πύλες" (gates). Τα GRUs χρησιμοποιούν 2 πύλες: update gate, reset gate. Τα LSTMs χρησιμοποιούν 3 πύλες: update gate, forget gate, output gate.
* **Εφαρμογές**: Natural Language Processing (NLP), Sentiment Analysis, DNA sequence classification, Speech recognition, Language Translation.

### 3. Convolutional Neural Networks (CNNs - Συνελιξιακά)
* Είναι εμπνευσμένα από την οργάνωση των νευρώνων στον οπτικό φλοιό (visual cortex) του εγκεφάλου. * Είναι ιδανικά για επεξεργασία δεδομένων όπως εικόνες, βίντεο και ήχος.
* Τα κρυφά επίπεδα σε ένα CNN αποτελούνται συνήθως από: Convolutional layers $\rightarrow$ Pooling layers $\rightarrow$ Fully-connected $\rightarrow$ Normalization.
* **Convolution (Συνέλιξη)**: Επιτρέπει την εξαγωγή οπτικών χαρακτηριστικών (visual features) σε κομμάτια (chunks). Μαθηματικός τύπος: $(f*g)(i) = \sum_{j=1}^{m} g(j)\cdot f(i-j+m/2)$ (Είναι ουσιαστικά το εσωτερικό γινόμενο - Dot product - της εισόδου με τον πυρήνα/kernel).
* **Pooling**: Μειώνει τον αριθμό των νευρώνων που είναι απαραίτητοι στα επόμενα επίπεδα. Υπάρχουν 2 τύποι: Max & Min pooling.
* **Ροή για Image classification (όπου τα CNNs κυριαρχούν)**: 
  1. Λαμβάνουμε την εικόνα εισόδου (2D μήτρα από pixels, τυπικά RGB).
  2. Κάνουμε συνέλιξη της εικόνας με πολλαπλά φίλτρα (δημιουργώντας ένα feature matrix output για κάθε φίλτρο).
  3. Κάνουμε pool τα αποτελέσματα για να παράγουμε μια υποδειγματοληπτική (downsample) μήτρα χαρακτηριστικών για κάθε φίλτρο στο convolution layer.
  4. Τυπικά επαναλαμβάνουμε τα βήματα 2 & 3 πολλαπλές φορές χρησιμοποιώντας τα προηγούμενα χαρακτηριστικά ως είσοδο.
  5. Προσθέτουμε λίγα fully-connected κρυφά επίπεδα για να βοηθήσουν στην ταξινόμηση της εικόνας.
  6. Παράγεται η πρόβλεψη ταξινόμησης στο επίπεδο εξόδου (output layer).
* **Εφαρμογές**: Computer vision, Image recognition, Image processing, Image segmentation, Video analysis.

## Βήματα Δημιουργίας ενός Deep Learning Model

1. **Συλλογή Δεδομένων (Gathering Data)**
   * Η επιλογή των σωστών δεδομένων είναι το κλειδί (Picking the right data is key).
   * Κάνε υποθέσεις για τα δεδομένα που χρειάζεσαι, κατάγραψέ τες και τέσταρέ τες αργότερα αν χρειαστεί.
   * **Μέγεθος**: Ποικίλλει, αλλά ένας εμπειρικός κανόνας είναι: Αριθμός δεδομένων = $10 \times$ Παράμετροι Μοντέλου. (π.χ. στο Regression: 10 παραδείγματα ανά μεταβλητή, στο Image-classification: ελάχιστο 1000 εικόνες ανά κλάση) .
   * **Ποιότητα**: Πόσο συχνά είναι τα λάθη ετικετοποίησης (labelling errors); Είναι τα χαρακτηριστικά σου "θορυβώδη" (noisy);.
   * Πηγές εύρεσης: ML Repository UCI, Kaggle, Google dataset search, Web scrapers (π.χ. Beautiful Soup).

2. **Προεπεξεργασία (Preprocessing the data)**
   * **a. Splitting (Διαχωρισμός)**: Χωρισμός του dataset σε υποσύνολα: Training, Validation, Test. Αν υπάρχουν λίγα hyperparameters, το validation set μπορεί να είναι μικρό. Συχνά χρησιμοποιούμε Cross-validation (k-fold). Για datasets με χρονοσειρές (Time-splitting), οι παλαιότερες μέρες πάνε στο train set και οι πιο πρόσφατες στο test set.
   * **b. Formatting**: Μετατροπή στο σωστό σχήμα (π.χ. CSV).
   * **c. Missing Data (Ελλιπή δεδομένα)**: Απαιτεί προσοχή και χρόνο. Αντιπροσωπεύονται συνήθως ως NaN ή Null. Αντιμετώπιση:
     1. Εξάλειψη χαρακτηριστικών με τιμές που λείπουν (Eliminating).
     2. Συμπλήρωση (Imputing) των τιμών.
   * **d. Sampling / Imbalanced Data (Μη ισορροπημένα δεδομένα)**: Όταν μια κλάση (Majority) υπερτερεί συντριπτικά της άλλης (Minority). Εφαρμόζουμε τεχνικές όπως το **downsampling** και το **upweighting** για να μη γίνει το μοντέλο "προκατειλημμένο" (biased) προς την πλειοψηφούσα κλάση.
   * **e. Feature Scaling (Κλιμάκωση χαρακτηριστικών)**:
     1. Normalization (min-max scaling σε κάθε στήλη χαρακτηριστικού).
     2. Standardization.

3. **Εκπαίδευση του μοντέλου (Training the model)**
   * Feed data $\rightarrow$ Forward Propagation $\rightarrow$ Υπολογισμός Loss function $\rightarrow$ Backpropagation.

4. **Αξιολόγηση (Evaluation)**
   * Αξιολογούμε το μοντέλο στο validation set και τέλος στο test set (το οποίο προορίζεται να αναπαραστήσει πώς θα λειτουργήσει το μοντέλο στον πραγματικό κόσμο - real world).

5. **Βελτιστοποίηση (Optimizing)**
   * **a. Hyperparameter Tuning**: Αύξηση του αριθμού των εποχών (epochs), προσαρμογή του Learning Rate. Οι αρχικές συνθήκες παίζουν μεγάλο ρόλο στο τελικό αποτέλεσμα πολύπλοκων μοντέλων.
   * **b. Αντιμετώπιση Overfitting**: Συλλογή περισσότερων δεδομένων (Getting more data), Regularization, Dropout, Data Augmentation, Μείωση μεγέθους μοντέλου (Reducing Models).
